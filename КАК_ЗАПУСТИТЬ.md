# Как запустить сайт (clashaiii)

## 1. Открой терминал именно в папке проекта

Важно: текущая папка должна быть `clashaiii`, а не родительская.

**В Cursor / VS Code:**
- Меню **Terminal → New Terminal** (или `` Ctrl+` ``)
- Внизу в терминале проверь путь. Должно быть что-то вроде:
  ```
  C:\Users\s2401117\clashaiii>
  ```
- Если ты в `C:\Users\s2401117`, перейди в проект:
  ```
  cd clashaiii
  ```

## 2. Установи зависимости (если ещё не ставил)

**Если используешь pnpm:**
```bash
pnpm install
```

**Если используешь npm:**
```bash
npm install
```

## 3. Очисти кэш Next.js и запусти сервер

**Вариант A — через pnpm:**
```bash
rmdir /s /q .next
pnpm run dev
```

**Вариант B — через npm:**
```bash
rmdir /s /q .next
npm run dev
```

После запуска в консоли появится что-то вроде:
```
▲ Next.js 15.x.x
- Local:   http://localhost:6969
```
Открой в браузере: **http://localhost:6969**

---

## Если dev не работает — запуск через сборку (режим production)

Так сайт точно запустится, без devtools и без ошибок package.json:

```bash
cd clashaiii
rmdir /s /q .next
npm run start
```

Скрипт `start` сначала соберёт проект (`next build`), потом запустит сервер на порту 6969. Открой **http://localhost:6969**.

После изменений в коде нужно снова запустить `npm run start` (пересборка + запуск).

---

## Если снова ошибка про package.json в режиме dev

1. В папке `clashaiii` выполни:
   ```bash
   rmdir /s /q .next
   npm run dev
   ```
2. Либо запусти через Turbopack: `npm run dev:turbo`
3. Файл `C:\Users\s2401117\package.json` был перезаписан без BOM — если ты его менял, сохраняй в кодировке **UTF-8 без BOM**.

## Если нет pnpm

Установи npm-зависимости и запускай через npm:
```bash
npm install
npm run dev
```
Порт будет тот же: **6969**.
